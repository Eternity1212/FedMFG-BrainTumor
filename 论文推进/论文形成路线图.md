# 从当前项目形成论文的路线图

更新时间：2026-06-22

## 1. 我们这篇论文到底要讲什么

一句话定位：

> 本文研究真实多中心脑肿瘤 MRI 分类中同时存在模态缺失、2D/3D 数据形态差异和客户端模型异构时，如何在不共享原始医学影像的前提下进行稳定联邦学习。

建议论文题目：

> FedMFG: Modality-aware Fusion and Prototype-guided Head Aggregation for Heterogeneous Federated Brain Tumor Classification with Missing MRI Modalities

中文理解：

> FedMFG：面向缺失 MRI 模态和异构模型的脑肿瘤联邦分类方法。

## 2. 当前真正可讲的创新点

### 创新点 1：不是只处理缺失模态，而是处理“模态缺失 + 模型异构”

已有 FedAMM 主要讲任意缺失模态的脑肿瘤分割。我们的差异要明确落在：

- 任务是多中心脑肿瘤分类，不是分割。
- 客户端既有 3D MRI，也有 2D MRI。
- 不同客户端的特征提取器结构不同，不适合做完整模型参数平均。
- 因此我们把联邦协作从“平均全模型”转为“共享高层语义原型 + 聚合分类头”。

### 创新点 2：模态感知融合，不伪造缺失模态

很多缺失模态方法会补零、生成缺失模态或估计缺失模态特征。我们的做法更保守：

- 有什么模态就用什么模态。
- 缺失模态不参与特征提取。
- 通过门控网络给可用模态分配权重。

这点适合医学场景，因为生成出来的医学影像不一定真实可靠。

### 创新点 3：模态组合级类别原型

普通 FedProto 只保存“类别原型”，例如 glioma 的平均特征。

FedMFG 保存“模态组合 + 类别”的原型，例如：

- `T1c + FLAIR` 下的 glioma 原型。
- `T1` 下的 glioma 原型。
- `T1 + T1c + T2 + FLAIR` 下的 glioma 原型。

这样可以避免把不同模态条件下的同类样本全部混在一起。

### 创新点 4：从更完整模态组合构造教师原型

如果某客户端只有单模态，它很难学到多模态语义。服务器可以用更完整模态组合的原型作为教师，约束少模态客户端的表示。

这可以写成：

> Full-modality or more-complete modality prototypes act as semantic teachers for incomplete-modality clients.

### 创新点 5：原型一致性指导分类头聚合

不同客户端数据质量、类别覆盖和模态完整度不同，上传的分类头并不一样可靠。

FedMFG 根据本地原型和全局原型的一致性来估计客户端可靠度，让更稳定的客户端对分类头贡献更大。

这是区别于 FedAvg、FedGH、FedProto、FedAMM 的重要点。

## 3. 当前结果怎么理解

毕业论文草稿中的主结果：

| 方法 | Accuracy | Macro F1 |
| --- | ---: | ---: |
| Solo | 84.25 | 84.05 |
| FD | 80.79 | 76.62 |
| FedGH | 73.18 | 68.28 |
| FedProto | 83.95 | 75.93 |
| FedTGP | 87.63 | 87.01 |
| FedMM | 78.58 | 77.50 |
| FedAMM | 85.91 | 85.27 |
| FedMFG | 90.25 | 90.29 |

这个结果说明：

- FedMFG 当前数字最好。
- 相比 FedAMM，Accuracy 高约 4.34 个百分点。
- 相比 FedTGP，Accuracy 高约 2.62 个百分点。
- Macro F1 与 Accuracy 接近，说明不是只靠大类样本提升。

但要注意：

- 仓库里的 `fedmfg_history.json` 还没有正式 test 结果。
- 部分训练曲线后期出现 NaN，需要排查稳定性。
- 目前结果更像“单次实验结果”，还不能直接支撑正式投稿。

## 4. 还缺什么才能成为论文

### 必须补 1：正式可复现结果

需要重新跑：

- `local`
- `fd`
- `fedgh`
- `fedproto`
- `fedtgp`
- `fedmm`
- `fedamm`
- `fedmfg`

每个算法至少 3 个随机种子，推荐 5 个随机种子。

最终论文表格要写：

```text
mean ± std
```

### 必须补 2：FedMFG 测试结果 JSON

当前最关键缺口是 `FedMFG` 没有完整 test JSON。需要保存：

- 总体 Accuracy
- 总体 Macro F1
- 总体 Loss
- 每客户端 Accuracy
- 每客户端 Macro F1
- 每客户端样本数

### 必须补 3：消融实验代码化

论文不能只写消融表，需要能跑出来。

建议增加参数：

- `--mfg_disable_modality_gate`
- `--mfg_disable_combo_prototype`
- `--mfg_disable_teacher`
- `--mfg_proto_lambda 0`
- `--mfg_head_lambda 0`
- `--mfg_head_weight_mode uniform`

### 必须补 4：缺失模态强度实验

参考 PmcmFL、MFCPL、FedAMM 的写法，增加不同缺失设置：

- 25% 缺失
- 50% 缺失
- 75% 缺失
- 单模态客户端比例变化
- 完整模态客户端数量变化

### 必须补 5：可视化

至少补：

- 主实验柱状图。
- 每客户端结果图。
- 混淆矩阵。
- t-SNE 或 UMAP 特征图。
- 模态门控权重可视化。
- 原型相似度热图。

### 必须补 6：数据集说明和统计

论文需要一张数据表：

| 客户端 | 数据来源 | 2D/3D | 模态 | 类别 | 训练样本 | 测试样本 |
| --- | --- | --- | --- | --- | ---: | ---: |

这张表非常重要，因为我们的创新点依赖数据异构设置。

## 5. 相关论文给我们的启发

### FedAMM 给我们的启发

链接：https://papers.miccai.org/miccai-2025/paper/1764_paper.pdf

我们要学习：

- 如何把“任意模态缺失”讲成医学场景痛点。
- 如何设计模态组合级实验。
- 如何强调服务器端原型对齐。

我们要避开：

- 不能只重复 FedAMM 的模态组合原型。
- 必须强调分类任务、2D/3D 异构和分类头可靠聚合。

### FedMEMA / FedMEPD 给我们的启发

链接：https://ojs.aaai.org/index.php/AAAI/article/view/27909  
链接：https://arxiv.org/abs/2603.04887

我们要学习：

- 怎样写 inter-modal heterogeneity。
- 怎样解释模态专属编码器。
- 怎样用全模态语义锚点帮助缺失模态客户端。

可转化到本项目：

- 把我们的 teacher prototype 写成“lightweight semantic anchor”。
- 用原型图证明少模态客户端被拉向更完整模态语义空间。

### PmcmFL 给我们的启发

链接：https://arxiv.org/abs/2312.13508

我们要学习：

- 原型库作为全局先验。
- 缺失模态比例变化实验。
- 训练阶段和推理阶段都缺失时的鲁棒性评估。

可转化到本项目：

- 增加单模态推理实验。
- 对比“无 teacher prototype”和“有 teacher prototype”。

### MFCPL 给我们的启发

链接：https://arxiv.org/abs/2401.13898

我们要学习：

- 严重模态缺失实验。
- 跨模态原型对齐。
- 使用原型缓解非 IID 和缺失模态同时存在的问题。

可转化到本项目：

- 增加 severe missing setting。
- 做原型相似度热图，证明不同模态组合被对齐。

## 6. 数据集应该怎么推进

### 第一阶段：复现当前数据

先把当前已有数据整理到：

```text
data/processed/
  BraTS/
  Shanghai/
  Figshare/
  Brisc2025/
```

然后把脚本里的：

```text
/data/zikun_workspace/preprocessed
```

改成项目内相对路径：

```text
../data/processed
```

### 第二阶段：补公开数据

优先补 Figshare，因为它公开、任务匹配、体积相对可控。

下载脚本已放在：

```text
data/scripts/download_figshare.sh
```

BraTS/FeTS 要走官方授权或 Kaggle/Synapse 登录，不能保证匿名脚本直接下载。

### 第三阶段：补预处理脚本

需要新增：

- `preprocess_figshare.py`
- `preprocess_brats.py`
- `build_federated_splits.py`
- `summarize_dataset.py`

目标是从原始数据自动生成代码可用的 `.npz` 文件。

## 7. 最推荐的执行顺序

### 第 1 周：把仓库变成可复现实验工程

1. 修正训练脚本数据路径。
2. 补数据统计脚本。
3. 补正式测试导出脚本。
4. 重新跑 FedMFG 单种子。

### 第 2 周：补主实验

1. 跑所有 baseline。
2. 跑 FedMFG。
3. 统一生成主结果表。
4. 检查 NaN 和训练稳定性。

### 第 3 周：补强实验

1. 多随机种子。
2. 消融实验。
3. 缺失模态强度实验。
4. 每客户端结果。

### 第 4 周：补论文图和初稿

1. 画混淆矩阵。
2. 画 t-SNE/UMAP。
3. 画原型相似度热图。
4. 写 introduction、method、experiment。

## 8. 最终投稿策略

如果只完成单种子和基本对比：

- 适合毕业论文。
- 适合中文会议、课程项目、workshop 初稿。

如果完成多种子、消融、缺失强度、可视化：

- 可以尝试 BIBM、ISBI、MICCAI workshop。
- 可以考虑 Computer Methods and Programs in Biomedicine、Artificial Intelligence in Medicine 等期刊。

如果还想冲 MICCAI/AAAI/TMI/MIA：

- 需要更强理论动机。
- 需要更大规模公开数据。
- 需要与 FedAMM/FedMEMA/PmcmFL/MFCPL 更严格复现对比。
- 需要稳定超过现有 SOTA，而不是只在自建混合场景领先。
